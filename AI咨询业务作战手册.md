### 1.2 脉脉/LinkedIn 内容素材（连续发5天）

#### 第1天 - 自我介绍篇
```
从小米出来后，最大的感受是：

大厂和中小企业在AI上的差距，根本不是模型，而是认知差。

大厂里有专门的团队做AI落地，知道哪件事做了能立刻见效。

中小企业里大家都在说AI，但90%的人不知道从哪下手。

不是买不起大模型，不是技术不够先进。

是没人告诉他们：你们公司这3件事，用AI现在就能做。

接下来打算把我在小米做AI信息化的经验整理出来，帮一些需要的公司少走点弯路。

如果你的团队也在看AI，但不知道从哪下手，可以私信我聊聊。
```

#### 第2天 - 避坑篇
```
很多公司做AI，第一个坑就踩错了。

一上来就想搞个大模型，想做个大平台。

我在小米做AI的时候，我们一开始也想过搞大的，但后来发现最有效的方式反而是：

先找一个具体的业务场景，用AI把它做透。

比如营销部门写文案，原来10个人写一天，用AI之后2小时就够了。
比如技术部门写代码，用AI之后效率能提升30%。

先把一件事做成，看到实实在在的效果，大家才会相信AI，才愿意继续投入。

中小企业做AI，不要先买大模型，先找一个具体的场景，把它做透。

这是我在小米踩了很多坑之后得出的结论。
```

#### 第3天 - 行业案例篇
```
上周和一个汽车行业的技术总监聊天。

他说他们团队也想做AI，但不知道从哪开始。

我问他：你们现在最头疼的是什么？

他说：要给主机厂做方案，每次写方案都要花很多时间。

我说：那你就先用AI写方案啊。

他恍然大悟。

很多时候不是AI不能用，是你没把AI和你最痛的那个点结合起来。

这就是为什么很多公司做AI的现状：人人都在说AI，但没有人把AI用在自己最痛的地方。
```

#### 第4天 - 认知篇
```
我发现一个很有意思的现象：

越懂技术的公司，越不会被AI的各种概念忽悠。

他们不会去管什么AGI，什么大模型排行榜。

他们只关心：这个东西能不能帮我把这件事做了，能不能省钱，能不能赚钱。

反而是不太懂技术的老板，最喜欢听各种概念，最后钱花了不少，什么实际效果都没有。

AI不是玄学，AI是工具，工具就要拿来用的，不是拿来说的。
```

#### 第5天 - 人才篇
```
很多人问我：中小企业做AI，需要招什么样的人？

我的答案是：不用招人。

至少一开始不用。

你不需要一个AI算法工程师，不需要一个专门的AI团队。

你需要的是：
1. 一个懂你业务的人
2. 知道AI现在能做什么，不能做什么
3. 选1-2个具体的场景，先做起来

等你看到效果了，再考虑招人也不迟。

我见过太多公司一上来就招了一堆人，最后什么都没做出来。
```

## 2. 客户跟进SOP - 有人来问之后怎么聊

### 2.1 客户说"感兴趣，具体怎么弄？"之后的完整流程

#### 第一步：先了解基本情况（不要一上来就报价）
```
太好了！我先简单了解一下你们的情况：

1. 你们公司大概多少人？主要做什么业务？
2. 现在团队里有没有人已经在用AI了？比如ChatGPT、Claude这些？
3. 你最希望用AI解决什么问题？或者说你最痛的点是什么？

了解完这些我给你说说这个Workshop具体怎么弄，以及对你们来说值不值。
```

#### 第二步：根据他的回答，定制化介绍Workshop
```
明白了，你们这个情况其实特别适合做这个Workshop。

我这个一天的Workshop，核心就是解决一个问题：**你们公司的AI到底应该从哪开始？**

不是讲AI理论，不是讲大模型原理，就是实打实的：
- 上午我给你们讲AI现在在你们这个行业，别人都在用它做什么，哪些是真有用，哪些是忽悠
- 下午我带着你们团队一起讨论，把你们的业务场景一个个过，找出3个最适合先做的
- 最后我们一起排出一个优先级，谁来做，多久做出来，预期能有什么效果

一天结束，你们拿到的不是一堆PPT，是一份**可以立刻执行的《AI落地行动清单》**。
```

#### 第三步：报价和打消顾虑
```
收费是8000块一天，我去你们公司现场做，大概5-10个人参加最合适。

你可能会想，8000块一天值不值？

我给你算笔账：
- 如果我们梳理出来一个场景，能让你们团队一个人的效率提升30%，那一年就能省好几万
- 如果我们找到一个场景，能用AI帮你们多赚钱，那个价值就更大了
- 退一万步说，就算最后只发现一个地方能用，这8000块也早就赚回来了

而且我在小米做了那么久AI信息化，踩过的坑都可以直接告诉你，你们不用再踩一遍。

你看你们最近哪天方便？我可以把详细的大纲发给你看看。
```

## 3. Workshop交付SOP - 收钱之后怎么讲

### 3.1 完整的一天时间安排

| 时间 | 内容 | 目标 |
|------|------|------|
| **9:00 - 9:30** | 开场和破冰 | 了解每个人的角色和他们最关心的问题 |
| **9:30 - 10:30** | AI现在到底能做什么 | 建立认知，打破信息差 |
| **10:30 - 12:00** | 行业案例和你们的机会 | 结合你们行业，讲具体怎么做 |
| **12:00 - 13:30** | 午饭 | 轻松聊天，了解更多情况 |
| **13:30 - 15:00** | 你们业务场景头脑风暴 | 每个部门提1-2个可能的场景 |
| **15:00 - 16:30** | 优先级排序和行动方案 | 选出Top 3，制定具体执行计划 |
| **16:30 - 17:00** | 总结和答疑 | 输出行动清单，解答最后问题 |

---

### 3.2 每一部分的详细内容

#### 第一部分：AI现在到底能做什么（9:30-10:30）
```
不讲任何原理，只讲实实在在能用的：

1. 内容生产类
   - 写方案、写报告、写邮件、写PPT大纲
   - 翻译、润色、改写、缩写、扩写
   - 生成营销文案、社交媒体内容

2. 技术研发类
   - 写代码、改代码、解释代码
   - 查bug、写测试用例、写文档
   - 架构设计讨论、技术选型分析

3. 数据分析类
   - 整理Excel数据、生成分析报告
   - 数据可视化、趋势预测
   - 客户数据分析、销售数据分析

4. 客户服务类
   - 自动回复客户问题
   - 生成客服话术
   - 客户反馈分析

每一类都给1-2个你们行业的具体例子。
```

#### 第二部分：行业案例和你们的机会（10:30-12:00）
```
结合客户的具体行业讲：

比如客户是汽车营销行业：
- 案例1：某汽车营销公司用AI写主机厂方案，效率提升3倍
- 案例2：某团队用AI生成汽车宣传文案，原来3个人的活现在1个人做
- 案例3：用AI分析用户评论，找出产品改进点

然后问：你们觉得这些里面，哪一个和你们现在的情况最像？
```

#### 第三部分：业务场景头脑风暴（13:30-15:00）
```
让每个部门的人都发言：
1. 你现在做的工作里，哪部分最花时间？
2. 你觉得哪部分可能能用AI做？
3. 如果这部分能用AI做，能省多少时间？

把所有想法都写在白板上，不要评判，先全部列出来。

目标：至少列出来10个可能的场景。
```

#### 第四部分：优先级排序和行动方案（15:00-16:30）
```
用三个维度给每个场景打分（1-5分）：
1. 见效速度：多快能看到效果？
2. 实施难度：技术上好不好做？需要多少人？
3. 业务价值：做了之后能省多少钱/赚多少钱？

然后选出得分最高的3个，给每一个都制定：
- 具体要做什么
- 谁来负责
- 多久能做完
- 预期达到什么效果

这就是最后的《AI落地行动清单》
```

## 4. 常见问题FAQ - 所有可能的问题标准答案

### Q1: 你们这个Workshop和其他AI培训有什么区别？
```
最大的区别是：别的培训讲的是"AI是什么"，我讲的是"你的公司的AI应该怎么用"。

- 不是讲大模型原理，不是讲AI发展史
- 所有案例都是我在小米实际做过的，或者我见过的其他公司做过的
- 不是我一个人讲一天，是一半时间我讲，一半时间带着你们团队一起讨论你们自己的业务
- 最后拿到的不是一堆PPT，是可以立刻执行的行动清单

简单说：别的培训是让你"知道AI"，我的Workshop是让你"用上AI"。
```

### Q2: 一天时间够吗？会不会太赶了？
```
其实一天刚刚好。

因为我们不需要讨论所有的可能性，我们只需要找出3个最适合你们先做的。

而且最重要的是：
- 上午讲完，大家建立了共同的认知
- 下午趁热打铁立刻讨论，效果最好
- 如果拖成两天，大家的思路就断了，反而效果不好

我做过好几次了，一天时间刚好够把这件事说透，并且拿出一个具体的结果。
```

### Q3: 我们团队人比较多，可以多来几个人吗？
```
建议5-10个人效果最好。

人太少的话，视角不够全面；人太多的话，讨论起来效率低，每个人发言的机会少。

最好的组合是：
- 1个技术负责人（你）
- 2-3个核心的业务负责人
- 3-5个一线的骨干员工

这样既有管理层的视角，也有一线执行的视角，最后出来的结果最容易落地。
```

### Q4: 可以线上做吗？
```
可以，但我强烈建议线下做。

因为线下讨论的效果是线上的好几倍：
- 大家坐在一起，更容易进入状态
- 白板讨论更方便，思路更清晰
- 中午吃饭的时候还能聊很多正式场合聊不到的东西

当然如果你们实在不方便，线上也可以，但是效果会打一些折扣。
```

### Q5: 做完Workshop之后，还有后续的支持吗？
```
有的。

做完Workshop之后，我会给你们留1个月的微信答疑时间，你们在执行过程中遇到任何问题都可以问我。

如果你们需要更深入的支持，比如帮你们把某个场景落地，或者做更长期的AI规划，我也有对应的咨询服务。

但首先，我们先把这一天的Workshop做好，找出那3个最该做的事，这是最重要的。
```

### Q6: 你对我们这个行业了解吗？
```
说实话，我可能不如你了解你们行业的具体业务。

但这恰恰是我的价值所在：
- 你懂你的业务，我懂AI能做什么
- 我们两个加在一起，就能找出最适合你们的方案
- 而且我做过很多不同行业的案例，很多方法其实是相通的

而且在来之前，我会先做功课，了解一下你们行业现在AI应用的情况。

所以你不用担心行业的问题，只要告诉我你们的业务痛点就行。
```
